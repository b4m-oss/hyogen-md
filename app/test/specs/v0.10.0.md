# v0.10.0 テスト仕様書

[roadmap v0.10.0](../../docs/roadmap.md) / [playground.md](../../docs/playground.md) の実装範囲に対応する。  
プレイグラウンド製品仕様は [playground.md](../../docs/playground.md) が正。  
本バージョンは **Playground UX のみ**（ライブラリ本体の API・パイプラインは変更しない）。

## スコープ

- ファイル操作 UI をスリードット（⋯）のアクションメニューへ移す
- `@hg` … `@endhg` / `@@` 内のシンタックスハイライト（JS 近似）。簡易な `{{ }}` 見やすさ
- 非エントリファイル（layout 等）のライブラリ ParseError を Diagnostics **note** に落とす（赤エラーと区別）。Preview はソース Markdown
- 純ロジックの単体テスト（メニュー項目決定、hyogen 領域検出、soft diagnostics）
- UI・ハイライト見た目は **手動確認**

## スコープ外（v0.10.0）

- Vue コンポーネントの自動 E2E / マウントテスト
- VS Code / 他エディタ向けプラグイン
- 専用 hyogen DSL 文法の完全ハイライト（`each` 等を JS 以外として厳密に色分け）
- データソース import / TOC / 許可メソッド（後続バージョン）
- outDir 一括ビルド、context 編集 UI

## Phase 1: アクションメニュー（純ロジック）

### menuItemsForNode（名前は実装で固定）

- ノード種別（`src-root` / `directory` / `file`）とパスから、表示するメニュー項目の配列を返す
- 副作用なし。OUT 用は空配列（または呼び出し側でメニュー非表示）

#### テスト: 正常系

- Given `src-root`（`/src`）→ Then 項目は New file, New folder のみ（Rename / Delete 無し）
- Given SRC `directory`（例: `/src/partials`）→ Then New file, New folder, Rename, Delete
- Given SRC `file`（例: `/src/index.md`）→ Then Rename, Delete のみ
- Given OUT パス → Then 空（メニュー無し）

#### テスト: 異常系

- Given `/src` を directory 扱いで渡した場合 → Rename / Delete を出さない（src-root と同じ扱い、または呼び出し側で src-root に正規化。テストで固定）
- Given 空パス → 空配列または throw（実装で固定しテストでロック）

## Phase 2: hyogen 領域検出（純ロジック）

### findHyogenRegions

- Markdown ソース文字列を受け取り、ハイライト対象の半開区間 `{ from, to, kind }` の配列を返す
- `kind` は少なくとも `hg-block`（`@hg`…`@endhg`）と `hg-short`（`@@`…`@@`）を区別してよい（実装で固定）
- **コードフェンス**（`` ``` ``）内に現れる hyogen は **含めない**
- HTML コメントの外側にある地の文は対象外（ブロック内部の DSL 本文のみ、またはコメント全体のうち DSL 部分。実装で固定しテストでロック）。推奨: ハイライトしやすいよう **コメント内の `@hg` 〜 `@endhg`（マーカー含むまたは内側のみ）** を区間にする

#### テスト: 正常系

- Given 通常の複数行 `<!-- @hg … @endhg -->` が 1 つ → Then 区間 1 件。フェンス外
- Given `<!--@@ const x = 1 @@-->`（または同等ショートハンド）→ Then 区間 1 件（`hg-short`）
- Given 文書に通常ブロックが 2 つ → Then 区間 2 件（出現順）
- Given フェンス内にだけ `@hg` がある → Then 区間 0 件
- Given フェンス外に 1 つ、フェンス内に 1 つ → Then フェンス外のみ 1 件
- Given hyogen 無しの素の Markdown → Then 空配列

#### テスト: 異常系

- Given 未閉じ `@hg`（`@endhg` 無し）→ 区間に含めない、または文書末まで（実装で固定。推奨: **不完全は捨てる**）
- Given 空文字 → 空配列

### findMustacheRegions（任意・同ファイルでも可）

- フェンス外の `{{ ... }}` / `{{{ ... }}}` の区間を返す（ネストは最小実装で可）
- コードフェンス内は含めない

#### テスト: 正常系

- Given `Hello {{ name }}` → Then 1 件
- Given フェンス内の `{{ name }}` のみ → Then 0 件

#### テスト: 異常系

- Given 未閉じ `{{` → 捨てる（推奨）

## Phase 3: 手動確認（UI）

自動テスト対象外。実装後に人手で確認する。

### アクションメニュー

- [ ] 上部の `+ file` / `+ folder` / `rename` / `delete` ボタン列が無い
- [ ] `src` 見出しの ⋯ から New file / New folder ができ、作成先が `/src` 直下になる
- [ ] SRC フォルダの ⋯ から、そのフォルダ配下に New file / New folder ができる
- [ ] SRC ファイル／フォルダの ⋯ から Rename / Delete ができ、**そのノード**が対象になる（別ノードを選択していてもメニュー元が優先）
- [ ] OUT ツリーに変更用 ⋯ が出ない（または操作できない）
- [ ] `/src` ルートに Rename / Delete が出ない

### シンタックスハイライト

- [ ] デモ `index.md` の `@hg` ブロック内が、地の HTML コメント一色ではなくキーワード／文字列等が色分けされる
- [ ] `@@` ショートハンドを書いたときも内側が色分けされる
- [ ] コードフェンス内に書いた `@hg` 風テキストはハイライトされない（または Markdown のコードとして扱われる）
- [ ] `{{ ... }}` が周囲の本文より識別しやすい（簡易で可）
