# v0.3.0 テスト仕様書

[roadmap v0.3.0](../../dev-docs/roadmap.md) の実装範囲に対応する。製品仕様は [dev-docs/specs/](../../dev-docs/specs/) が正。

## スコープ

- hyogen ブロック内 **宣言・代入**（`const` / `let`、配列・オブジェクトリテラル、再代入）
- **if / else if / else / endif**（Pug 風、pipeline step 5 展開）
- **each / endeach**（Pug 風、pipeline step 5 展開）
- 条件式・ループ式は **`{{ }}` と同じ式サブセット**（v0.3 で比較・論理・単項 `!`・算術を追加）
- **`if` / `each` 構造ネスト合計 20** → `nest_limit_exceeded` **警告** + 当該ブロックスキップ
- パイプライン: **宣言（step 2）→ include（step 4）→ if/each（step 5）→ `{{ }}`（step 6）**
- v0.2 回帰（194 テスト）維持

## スコープ外（v0.3.0）

- 三項演算子（`{{ a ? b : c }}`）— [roadmap v0.7](../../dev-docs/roadmap.md)
- `for` / `do…while` — v0.7
- `extend` / `block` — v0.4
- `@@` ショートハンド — v0.7
- コードフェンス内 hyogen ブロック — 引き続き無視（[dsl.md](../../dev-docs/specs/dsl.md)）

## 確定事項（v0.3.0）

| 項目 | 確定内容 |
|------|----------|
| パイプライン順 | `executeDeclarations` → `executeHgBlocks` → `expandIncludes` → `expandControlStructures` → `interpolateExpressions` → `stripHgComments` |
| 宣言ブロック | **複数行** `@hg` ブロック。`const` / `let` / 再代入のみ。実行後 **ブロックごと除去** |
| 構造制御ブロック | **1 行** `@hg` ブロック（`if` / `else if` / `else` / `endif` / `each` / `endeach`）。step 2 では **触らず残す**（pass-through） |
| `executeHgBlocks` | include / component は従来どおり。構造制御 1 行ブロックは **エラーにしない**（ソースに残す） |
| 条件評価 | JS の **truthy / falsy**（`0` / `""` / `null` / `undefined` / `false` / `NaN` は falsy） |
| each イテレーション | ループ変数名（例: `item`）を **scoped context** に束縛。本体は `expandControlStructures` + **部分 `interpolateExpressions`** |
| if 分岐 | **本文テキスト選択のみ**（step 5）。分岐外・分岐内の残り `{{ }}` は全体 step 6 |
| ネスト上限 | `if` / `each` の **構造ネスト合計** 最大 20。21 段目以降 → **`nest_limit_exceeded` 警告 + スキップ**（中断しない） |
| include 子 | `renderChildDocument` → `renderDocumentBody` 経由で **宣言・if/each も実行** |
| 予約語 | [dsl.md 予約語](../../dev-docs/specs/dsl.md) を識別子に使用 → **`parse_error`** |
| 危険キー | 宣言・式評価時に **`forbidden_property_access`**（v0.2 継続） |
| 演算子優先順位（高→低） | 単項 `!` → 比較（`===` `!==` `==` `!=` `>=` `<=` `>` `<`）→ 論理（`&&` `||`）→ デフォルトパイプ `\|` → メンバ `.` / コール |
| 算術 | `+` `-` `*` `/` を条件式・宣言右辺で許可（比較より **高** 優先。例: `1 + 2 >= 3`） |
| `let` 再代入 | **許可**（同一 `@hg` ブロック内および後続宣言ブロック） |
| `const` 再代入 | **拒否** → **`parse_error`** |
| 空分岐 | 本文なしの `if` / `else` / `each` イテレーション → **空文字出力**（エラーにしない） |
| 構文エラー | 未対 `if`、余分な `endif` / `endeach`、`else` のみ、`if` 無し `endif` 等 → **`parse_error`** |
| strip | 展開後、`if` / `else if` / `else` / `endif` / `each` / `endeach` の hyogen コメントは step 7 で除去 |

### レビュー待ち（プレースホルダ）

| 項目 | 候補 / メモ |
|------|-------------|
| 宣言ブロック + include 同居 | **不可**（宣言は複数行ブロック、include は 1 行ブロック — 別ブロック前提） |
| 同一行 `if` + 本文 minify | dsl 例どおり `<!-- @hg\nif x\n@endhg-->` 形式を主テスト。`@@` は v0.7 |

## Phase 1: 式エンジン拡張

対象: [parseExpression.ts](../../src/expr/parseExpression.ts) / [evaluateExpression.ts](../../src/expr/evaluateExpression.ts)

v0.2 の識別子・リテラル・`.`・`\|`・component call・`.toLocaleString` に加え、比較・論理・単項 `!`・算術を追加する。

### parseExpression（演算子）

#### テスト: 正常系

- `true` / `false` / `null` / `123` / `"x"` — 従来どおり
- `score >= 90` → 比較ノード
- `a === b` / `a !== b` / `a == b` / `a != b` / `a <= b` / `a > b` / `a < b`
- `active && score >= 90` → 論理 AND + 比較のネスト
- `a || b` / `!flag` / `!(a && b)`
- `1 + 2 * 3` → 算術優先（`*` > `+`）
- `count >= 1 + 2` → 算術が比較より先
- `title | "Untitled"` — デフォルトパイプは論理より **後**（`a || b | "x"` の結合をテストで固定）
- 既存 v0.2 式（`greet({ name: "Ada" })`、`meta.id`、`color | "transparent"`）が引き続き parse 可能

#### テスト: 異常系

- 空式 → **`parse_error`**
- 未閉じ括弧 `(a && b` → **`parse_error`**
- 未知演算子 `a <> b` → **`parse_error`**
- 三項 `a ? b : c` → **`parse_error`**（v0.3 スコープ外）

### evaluateExpression（演算子）

#### テスト: 正常系

- 比較: `90 >= 90` → `true`、`1 == "1"` → `true`（緩い等価）
- 論理: `true && false` → `false`、`false || "ok"` → `"ok"`（短絡は **評価結果のみ** 一致即可。短絡副作用テストは不要）
- 単項: `!0` → `true`、`!!""` → `false`
- 算術: `1 + 2` → `3`、`10 / 4` → `2.5`
- ネスト: context `{ score: 95, active: true }` で `active && score >= 90` → `true`
- デフォルトパイプ + 比較: `count | 0` が falsy 扱いになること（if 条件で使用）

#### テスト: 異常系

- `__proto__` メンバ → **`forbidden_property_access`**
- 未登録 component 呼び出し → **`parse_error`**（v0.2 継続）

## Phase 2: 宣言実行（step 2）

新規 `app/src/logic/`:

| 関数 | 役割 |
|------|------|
| `parseDeclaration` | 1 文（`const x = ...` / `let y = ...` / `x = ...`）を解析 |
| `executeDeclaration` | context へ束縛または再代入 |
| `executeDeclarations` | 文書中の宣言 `@hg` ブロックを **出現順**に実行し除去 |

### parseDeclaration

#### テスト: 正常系

- `const text = "foo bar"`
- `let fooBar = "over-writable"`
- `fooBar = "updated"`（再代入）
- `const data = ["apple", "banana"]`
- `const object = { animal: "duck", state: "sitting" }`
- 末尾セミコロンあり・なし両方
- `//` 行コメント・`/* */` ブロックコメントをスキップして解析

#### テスト: 異常系

- `const` なし裸代入で未宣言変数 → **`parse_error`**
- `const x = 1` 後 `x = 2` → **`parse_error`**
- 予約語 `const if = 1` → **`parse_error`**
- 空文 → **`parse_error`**
- `include ./x.md` 形式 → 宣言パーサ対象外（別処理）

### executeDeclaration / executeDeclarations

#### テスト: 正常系

- `const` 実行後 context に値が入る
- `let` 再代入が context を更新
- 複数文を順に実行（dsl 定数・変数例どおり）
- 複数の宣言 `@hg` ブロックを **文書出現順**に処理
- 実行後、宣言ブロックがソースから **除去**される
- front matter context と **マージ**（宣言が上書き可能 — テストで 1 ケース固定）

#### テスト: 異常系

- 右辺式 parse 失敗 → **`parse_error`**
- 右辺で危険キー → **`forbidden_property_access`**
- 未対 `@hg`（`@endhg` 欠落）→ **`parse_error`**

### executeHgBlocks（pass-through 修正）

#### テスト: 正常系

- 1 行 `if isNight` ブロック → ソースに **残る**（除去しない）
- `else` / `else if score >= 60` / `endif` / `each item in data` / `endeach` も同様
- include / component は v0.2 どおり動作

#### テスト: 異常系

- 複数行ブロックで include 以外 → 従来 **`parse_error`**（宣言ブロックは **先に** `executeDeclarations` で除去済み）
- 未対 `@hg` → **`parse_error`**

## Phase 3: 構造パーサ

新規 `app/src/control/`:

| 関数 | 役割 |
|------|------|
| `parseControlOpener` | 1 行 `@hg` 内から制御命令を判定 |
| `parseControlStructures` | ソース走査 → if/each ツリー（AST） |

### parseControlOpener

#### テスト: 正常系

- `if isNight` → `{ kind: "if", expr: ... }`
- `else if score >= 60` → `{ kind: "else_if", expr: ... }`
- `else` → `{ kind: "else" }`
- `endif` → `{ kind: "endif" }`
- `each item in data` → `{ kind: "each", item: "item", expr: ... }`
- `endeach` → `{ kind: "endeach" }`
- 前後空白・大文字小文字（キーワードは **小文字想定**。`IF` は **`parse_error`**）

#### テスト: 異常系

- `if` のみ（式なし）→ **`parse_error`**
- `each item data`（`in` 欠落）→ **`parse_error`**
- `each in data`（変数名欠落）→ **`parse_error`**
- include 行を渡す → 制御命令ではない（`null` または判別可能な結果 — 実装方針をテストで固定）

### parseControlStructures

#### テスト: 正常系

- `if` … 本文 … `endif` の単一ブロック
- `if` … `else if` … `else` … `endif` 連鎖
- `each item in expr` … 本文 … `endeach`
- 入れ子: `if` 内 `each`、`each` 内 `if` を AST で表現
- 空本文（opener と closer が隣接）も valid

#### テスト: 異常系

- `if` 無しで `endif` → **`parse_error`**
- `if` 開始後 `endif` 欠落 → **`parse_error`**
- `else` が `if` より前 → **`parse_error`**
- `else if` が `if` 無し → **`parse_error`**
- `each` 無しで `endeach` → **`parse_error`**
- `each` 開始後 `endeach` 欠落 → **`parse_error`**

## Phase 4: 構造展開 + ネスト上限

| 関数 | 役割 |
|------|------|
| `StructureNestTracker` | 深さカウント。20 超で skip 判定 |
| `expandControlStructures` | AST に従い Markdown 生成（async、warnings 集約） |

### StructureNestTracker

#### テスト: 正常系

- 深さ 0 → enter → 1 → exit → 0
- 深さ 20 まで enter 可能
- if と each は **同じカウンタ**に加算

#### テスト: 異常系

- 21 段目 enter → `{ exceeded: true, limit: 20 }`（警告用。throw しない）

### expandControlStructures

#### テスト: 正常系

- **if 真**: 第 1 分岐本文のみ出力（dsl Light Status 例）
- **if 偽 + else**: else 本文のみ
- **else if 連鎖**: 最初に真になった分岐のみ（grades fixture）
- **each 配列**: 3 要素 → 本文が 3 回連結（`- apple` 等）
- **each オブジェクト配列**: `item.key` / `item.value` がループごとに解決（scoped interpolate）
- **空 each**: `[]` → 空出力
- 入れ子 if/each が正しく再帰展開
- 展開後ソースに `@hg` 構造制御ブロックが **残る**（strip は step 7）

#### テスト: 異常系

- 深さ 21 のブロック → **スキップ**（空または原文削除 — 実装方針: **ブロック全体を空出力**）+ **`nest_limit_exceeded`** warning
- 条件式 parse 失敗 → **`parse_error`**
- `each` 式が非 iterable（`null` 等）→ **空出力**（警告なし — テストで固定）
