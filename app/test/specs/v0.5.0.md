# v0.5.0 テスト仕様書

[roadmap v0.5.0](../../dev-docs/roadmap.md) の実装範囲に対応する。製品仕様は [dev-docs/specs/](../../dev-docs/specs/) が正。

## スコープ

- **`build()`** — エントリ列挙 + glob → レンダリング → `outDir` へ書き出し（[api.md](../../dev-docs/specs/api.md)）
- **glob** — picomatch + fast-glob（Vite/JS エコシステム互換）
- **`_` partial 除外** — マッチ後フィルタ（glob 明示 / `includeUnderscoreEntries` で上書き可 → [paths.md](../../dev-docs/specs/paths.md)）
- **依存グラフ走査** — include / component / extend を静的収集（**出力はエントリのみ**。走査は検証・グラフ構築用）
- **`serverContext`** — `renderServer` / `build` のみ（別引数。template context にマージ）
- **リモート loader** — Node デフォルト loader が `http://` / `https://` を fetch（include / component / extend）
- **最小 `renderClient`** — export のみ。`serverContext` 渡し → **`server_context_on_client`**
- v0.4 回帰（319 テスト）維持

## スコープ外（v0.5.0）

- **`renderClient` 本実装**（loader 必須 CSR）— [roadmap v0.6](../../dev-docs/roadmap.md)
- **依存先を追加エントリとして build 出力** — entries-only
- **`suspicious_context_value`** — v0.7
- context 秘密キー名の機械検出 — [api.md](../../dev-docs/specs/api.md) 非ゴール

## 確定事項（v0.5.0）

| 項目 | 確定内容 |
|------|----------|
| 公開 API 追加 | `build(options)`, `renderClient`（stub）, `BuildOptions` / `BuildResult` 型 |
| デフォルト loader | Node: **`createNodeLoader()`** = FS + リモート fetch。`renderServer` / `build` の loader 省略時に使用 |
| outDir 映射 | **root からの相対パスを outDir にミラー**（`pages/a.md` → `outDir/pages/a.md`） |
| build 出力対象 | **glob/列挙で得たエントリのみ**。依存走査は検証・グラフ用（依存先は単体ファイルとして出力しない） |
| `_` 除外 | ファイル名 `_` 始まり、または `_` で始まるディレクトリ配下 → **エントリから除外** |
| `_` 上書き | input glob が `_partials/**` 等で **明示マッチ**したパス、または `includeUnderscoreEntries: true` → エントリに含める |
| glob 基準 | `options.root` または `.doc_root` 探索結果を **cwd** として fast-glob 実行 |
| 依存収集 | **静的**（render 前）。`scanHgBlocks` + 既存 parser（include / component / extend） |
| 依存グラフ | エントリから BFS/DFS。ノード = 絶対パス or URL。辿った依存は graph に含むが **出力しない** |
| 欠落依存 | build 前検証で `file_not_found` → **中断**（render 時と同型） |
| serverContext マージ | `mergeContext(context, serverContext)` — **serverContext 後勝ち**（[renderServer.ts](../../src/renderServer.ts) 既存順序） |
| serverContext 利用 | `renderServer` / `build` のみ。template の `{{ key }}` から参照可 |
| リモートパス | `isRemotePath(url)` = `http://` / `https://` 始まり。`resolveTemplatePath` **bypass** し loader に URL 直渡し |
| リモート + doc_root | URL は root 制約**外**。FS パスのみ `constrainToRoot` |
| renderClient stub | `serverContext` 非 null/undefined → **`server_context_on_client`**。それ以外は v0.6 まで未実装（専用 code または `parse_error` — テストで固定） |
| build 戻り値 | `{ files: { path, markdown }[], warnings }`。`path` は **outDir からの相対**または root 相対 — **root 相対で固定** |
| エントリ重複 | 同一相対パスが複数 input でマッチ → **後勝ち**（warning なし） |
| 書き出し | `fs/promises.mkdir` recursive + `writeFile`。親ディレクトリ自動作成 |
| extend + remote | v0.5 loader 拡張で **サポート**（v0.4 で deferred 分を含む） |

- `includeUnderscoreEntries: true` → `_` ファイルも glob マッチ分含む

#### テスト: 異常系

## Phase 5: build()

新規 `app/src/build/build.ts`

```ts
type BuildOptions = RenderOptions & {
  input: string | string[];
  outDir: string;
  includeUnderscoreEntries?: boolean;
  context?: HyogenContext;
  serverContext?: HyogenContext;
};

type BuildResult = {
  files: { path: string; markdown: string }[];
  warnings: HyogenWarning[];
};
```

### build

#### テスト: 正常系

- `input: ["pages/hello.md"]` → 1 ファイルレンダリング + `outDir/pages/hello.md` 作成
- **ミラー映射**: `pages/nested/a.md` → `outDir/pages/nested/a.md`
- `files[].path` は root からの相対パス
- `files[].markdown` はレンダリング結果（front matter strip デフォルト）
- 複数エントリ glob → 各ファイル出力
- include 先 partial は **outDir に単体出力されない**（entry の markdown に埋め込みのみ）
- warnings を全エントリ分集約
- `context` + `serverContext` 両方適用

#### テスト: 異常系

- エントリ 1 件が `parse_error` → **build 全体中断**（throw）
- `outDir` 書き込み不可 → **`load_failed`** または Node errno — **load_failed で固定**
- 依存 graph 上の欠落 → **`file_not_found`**
