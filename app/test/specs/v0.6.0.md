# v0.6.0 テスト仕様書

[roadmap v0.6.0](../../dev-docs/roadmap.md) の実装範囲に対応する。製品仕様は [dev-docs/specs/](../../dev-docs/specs/) が正。

## スコープ

- **`renderClient()`** — CSR 向け本実装（v0.5 stub 置換）
- **loader 必須** — 省略時は **`parse_error`**（デフォルト FS/fetch loader は付けない）
- **ライブラリはネットワーク取得しない** — 依存解決は注入 loader のみ（fetch / createNodeLoader を呼ばない）
- **同一オリジン想定** — CORS ランタイム強制なし。loader 契約として文書化
- **公開面分離** — `hyogen-md`（Node フル）と **`hyogen-md/client`**（ブラウザ安全）
- v0.5 回帰（374 テスト）維持

## スコープ外（v0.6.0）

- `@@` / for / 三項 / フェンス無視 — v0.7
- `build()` の変更 — Node エントリのみ（触らない）
- クロスオリジン拒否のランタイム強制 — loader 側の責務

## 確定事項（v0.6.0）

| 項目 | 確定内容 |
|------|----------|
| package exports | `"."` = Node フル、`"./client"` = ブラウザ向け |
| client export | `renderClient`, `createHyogenError`, `formatMessage`, 公開型のみ |
| client 非 export | `renderServer`, `build`, `createFsLoader`, `createNodeLoader`, `isRemotePath` |
| `{ path }` 入力 | FS の `resolveRenderInput` は使わない。`await loader(path)` でソース取得 |
| 文字列入力 | `renderDocument(source, …)` に直接。依存は loader 経由 |
| serverContext | **`server_context_on_client`**（loader チェックより先） |
| loader 未指定 | **`parse_error`**（`details.message` に loader required） |
| パス解決 | 既存 `resolveTemplatePath`。client ビルドでは `node:path` を **path-browserify** に alias |
| `path` / `root` 伝播 | `{ path }` 時の入力 path と `options.root` を `renderDocument` に渡す |

## Phase 1: renderClient 本実装

対象: [renderClient.ts](../../src/renderClient.ts)

#### テスト: 正常系

- 文字列 + mock loader（依存無し）→ Markdown 返却
- `{ path }` + loader が path を返す → 当該ソースをレンダリング
- context 変数が `{{ }}` に反映
- `options.root` / preserve オプションが伝播

#### テスト: 異常系

- loader 未指定 → **`parse_error`**
- `serverContext` あり → **`server_context_on_client`**（loader 有無に関わらず先）
