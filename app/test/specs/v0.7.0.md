# v0.7.0 テスト仕様書

[roadmap v0.7.0](../../docs/roadmap.md) の実装範囲に対応する。製品仕様は [docs/specs/](../../docs/specs/) が正。

## スコープ

- **`@@` ショートハンド** — `@hg` / `@endhg` と等価（[dsl.md](../../docs/specs/dsl.md)）
- **`for (init; cond; update)` / `do { … } while (cond)`** — hyogen ブロック内の実行文（step 2）。Pug 風 `each` とは別
- **三項演算子** — `{{ }}` / 式内（例: `isNight ? "Dark" : "Shine"`）
- **テンプレートリテラル** — `` `hello ${name}` ``。`${}` 内は許可式のみ、**component 呼び出し不可**
- **`.toLocaleString(...)`** — 既存実装の回帰固定（v0.2 済み）
- **コードフェンス内 hyogen 無視** — 既存実装を `@hg` / `@@` 両方で回帰固定
- **`suspicious_context_value`** — context 文字列リーフを検査、改変せず警告のみ（[security.md](../../docs/specs/security.md)）
- 付随: ループ用 **`++` / `--` / 複合代入（`+=` `-=` 等）**
- v0.6 回帰（380 テスト）維持

## スコープ外（v0.7.0）

- 単独 `while`、`for…of` / `for…in` — dsl 禁止
- ビルトイン関数の追加 — v0.8 以降
- mixin — 保留
- each 内 component 公式例のドキュメント追記 — v0.8

## 確定事項（v0.7.0）

| 項目 | 確定内容 |
|------|----------|
| `@@` 開始/終了 | `<!--@@` … `@@-->`。開始 `@@` と終了 `@@` のペア（`@endhg` 相当） |
| `@@` と通常形 | **等価**。混在（`@hg` … `@@` 等）→ **`parse_error`** |
| フェンス | `` ``` `` 内の `@hg` / `@@` ブロックは **無視**（実行・展開しない） |
| 三項優先順位 | 論理 `\|\|` の後、デフォルトパイプ `\|` の前（`a \|\| b ? c : d` → `(a\|\|b)?c:d`） |
| テンプレ `${}` | 同一式パーサ。component call 不可。`.toLocaleString` 可 |
| for | C 風のみ。本体は波括弧 `{ … }`。Markdown 本文を挟まない |
| for/do 配置 | **複数行** `@hg` / `@@` ブロック内（step 2） |
| ループ本体 | `const` / `let` / 再代入 / `++` `--` / 複合代入 / 入れ子 for・do。include / component / 構造 if・each は不可 |
| `++` / `--` | 識別子への前置・後置。for 更新節・ループ本体で許可 |
| 複合代入 | `+=` `-=` `*=` `/=`（識別子左辺） |
| suspicious 走査 | 宣言・文実行後の **最終 context** を再帰走査（文字列リーフ）。値は **改変しない** |
| suspicious reason | `contains_html_script_tag` / `contains_event_handler` / `contains_dangerous_scheme` / `contains_embed_tag` / `contains_meta_refresh`（[messages.en.json](../../docs/specs/messages.en.json)） |
| toLocaleString | 既存どおり。引数バリデーションなし |

## Phase 1: `@@` ショートハンド + フェンス回帰

対象: [scanHgBlocks.ts](../../src/parse/scanHgBlocks.ts)、[hgBlockUtils.ts](../../src/logic/hgBlockUtils.ts)

### scanHgBlocks / findUnclosedHgBlock

#### テスト: 正常系

- `<!--@@\ninclude ./a.md\n@@-->` → 1 ブロック検出
- 通常形 `@hg`…`@endhg` と同一ソース内容なら同等の `inner`（マーカー差のみ）
- フェンス内 `@@` / `@hg` → **検出されない**
- 既存フェンス内 `@hg` テストが Green

#### テスト: 異常系

- `@@` 開始のみで終了 `@@` 欠落 → `findUnclosedHgBlock` true / execute 段で **`parse_error`**
- `@hg` 開始 + `@@` 終了（混在）→ **`parse_error`**

### パイプライン等価

#### テスト: 正常系

- 同一 include / const を `@hg` と `@@` で書いた 2 ソース → **同じ markdown**

## Phase 2: 三項演算子 + テンプレートリテラル

対象: [parseExpression.ts](../../src/expr/parseExpression.ts)、[evaluateExpression.ts](../../src/expr/evaluateExpression.ts)、[types.ts](../../src/types.ts)

### parseExpression / evaluateExpression（三項）

#### テスト: 正常系

- `true ? "a" : "b"` → `"a"`
- `false ? "a" : "b"` → `"b"`
- `a || b ? c : d` の結合（`||` が先）
- `x ? y : z | "default"` — パイプが三項より後
- dsl 例: `isNight ? "Dark" : "Shine"`

#### テスト: 異常系

- `a ? b`（`:` 欠落）→ **`parse_error`**
- 旧テスト「三項は拒否」→ **許可に更新**

### テンプレートリテラル

#### テスト: 正常系

- `` `hello ${name}` `` → 連結文字列
- `` `${n.toLocaleString('ja-JP')}` `` → 許可
- 宣言右辺 `` const msg = `hi ${x}` `` → context に入る

#### テスト: 異常系

- `` `${greet({ name: "Ada" })}` `` → **`parse_error`**（component 不可）
- 未閉じバッククォート → **`parse_error`**

## Phase 3: for / do-while + 更新演算子

新規: `parseStatement` / `executeStatement` 等。`executeDeclarations` を文リスト実行に拡張。

### parseStatement / executeStatement

#### テスト: 正常系

- `for (let i = 0; i < 3; i = i + 1) { sum = sum + i }` → `sum === 3`（0+1+2）
- `for (;;)` の空 init/update 可（条件のみ）— または最小で init+cond+update 必須 — **三者とも空可、条件空は true 扱いだが無限ループ防止のため条件空は `parse_error`**
- `i++` / `++i` / `i += 2`
- `do { n = n + 1 } while (n < 3)`
- 入れ子 for

#### テスト: 異常系

- `while (x) { … }` 単独 → **`parse_error`**
- `for (x of y)` / `for (x in y)` → **`parse_error`**
- 未対 `{` → **`parse_error`**
- ループ内 `include` / `if` 構造 → **`parse_error`**

### executeDeclarations 統合

#### テスト: 正常系

- 複数行 `@hg` に const + for が混在 → 出現順実行後ブロック除去
- `@@` ブロックでも同様

#### テスト: 異常系

- for 条件式の parse 失敗 → **`parse_error`**
