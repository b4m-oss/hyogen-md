# v0.8.0 テスト仕様書

[roadmap v0.8.0](../../docs/roadmap.md) の実装範囲に対応する。製品仕様は [docs/specs/](../../docs/specs/) が正。

## スコープ

- **仕上げ・ドキュメント追随**（大規模な新規 DSL は無し）
- [need_decision.md](../../docs/need_decision.md) の未決を閉じる／v0.9+ へ明示転送
- **each 内 component** の公式例を [templating.md](../../docs/specs/templating.md) に追記 + E2E 固定
- **診断ログ** `formatDiagnosticLog` — [api.md](../../docs/specs/api.md) 例形式の公開ヘルパ
- **ビルトイン関数は導入しない**（判断を docs に記録）
- **suspicious 正規表現**を security.md に確定転記（v0.7 実装ベース）
- spec 各章の **代表統合テスト** + **警告は継続・エラーは中断** の回帰
- v0.7 回帰（423 テスト）維持

## スコープ外（v0.8.0）

- 新規 DSL 構文
- ビルトイン関数の実装
- mixin / HTML 出力レイヤ / パフォーマンス — v0.9+

## 確定事項（v0.8.0）

| 項目 | 確定内容 |
|------|----------|
| ビルトイン | **追加しない**。dsl.md / need_decision に「当面なし・将来候補」と明記 |
| each + component | ループ変数が component props に見える（既存挙動）。templating 公式例 + fixture |
| `formatDiagnosticLog` | `("error" \| "warning", diagnostic) => string`。ライブラリは **console 自動出力しない** |
| ログ 1 行目 | `[hyogen:error] {code}` または `[hyogen:warning] {code}` |
| ログ後続行 | `details` の各キーを `  {key}: {value}`（インデント 2 スペース）。`diagnostic.path` も `path` として含める |
| details の値 | 文字列化（`String(value)`）。undefined キーは省略 |
| suspicious 正規表現 | [scanSuspiciousContext.ts](../../src/security/scanSuspiciousContext.ts) のパターンを security.md に確定転記 |
| need_decision 残存 | mixin 保留 + 将来ビルトイン候補程度。実装待ちの未決は残さない |
| 警告継続 | `circular_include` / `prop_*` / `suspicious_context_value` / `nest_limit_exceeded` / `extend_in_component` → markdown 返却 + warnings |
| エラー中断 | `parse_error` / `file_not_found` / `forbidden_property_access` 等 → Promise reject、部分成功なし |
