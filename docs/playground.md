## 仮想 FS

| 項目 | 方針 |
|------|------|
| 実ディスク | **操作しない**。ブラウザ上のバーチャル FS のみ |
| 領域 | `src` と `outDir` を分離 |
| `src` | 編集可。新規作成・削除・リネーム・フォルダ作成可。`_` partial も **置ける**（include / component 参照用） |
| `outDir` | **読み取り専用**（展開結果の閲覧） |
| `_` partial と outDir | SSG / SSR のエントリ除外に **揃える**（[pipeline.md](./specs/pipeline.md)）。`_` で始まるファイル名、またはパス上いずれかのディレクトリ名が `_` 始まりのファイルは **outDir に書かない**／ツリーにも出さない。開いてプレビューすること自体は可（CSR 的な確認用） |
| 永続化 | `localStorage`（ブラウザ内のみ） |
| Reset | 「Reset to demo」でデモシードに戻し、localStorage も上書き |

## UI

| 項目 | 方針 |
|------|------|
| ファイラー | 左ペイン・縦方向。ディレクトリネスト可。`src` / `outDir` を区別して表示 |
| ファイル操作 UI | create / rename / delete は **優先度高**で、上部ボタン列ではなく **アクションメニュー**にする。`src` 見出し・各フォルダ名・各ファイル名の **右側にスリードット（⋯）** を置き、対象ノードに紐づく操作を出す（VS Code / Finder 寄り） |
| シンタックスハイライト | **優先度高・Playground 限定の重要機能**。`@hg` … `@endhg` および `@@` 内の DSL を、コメント一色ではなく読める色分けで見せる（書き方をその場でわかりやすくする）。詳細は下節 |
| プレビュー | **展開後 Markdown ソース**（標準 MD シンタックスハイライト）と **HTML 見た目**の両方（タブまたは分割） |
| HTML 化 | playground 側の責務（例: marked）。hyogen-md は Markdown 出力のみ |
| render タイミング | 入力のたびに自動（デバウンスあり） |
| outDir 更新 | **いま開いている 1 ファイルだけ**展開して対応パスへ書く（後から一括に変更しうる）。ただし `_` partial は上表のとおり **書かない** |
| 診断 | プレビュー付近にパネル。**本物のエラー**は赤系で目立たせ、警告は一覧。layout 等の「エントリではない」ためのライブラリ ParseError（例: orphan block without extend）は **note** 扱いでさりげなく出し、Preview は **ソースを Markdown として**表示（`/out` には書かない） |
| context | UI なし。シード／コード内の固定 context のみ |

### `@hg` / `@@` シンタックスハイライト（優先度高）

Playground のデモ・試用体験の中核。HTML コメント内の hyogen ブロックが地のコメント色のままでは、制御構文が追えない。

| 項目 | 方針 |
|------|------|
| 範囲 | **Playground（CodeMirror 6）のみ**。ライブラリ本体の責務ではない |
| 対象 | `@hg` … `@endhg`、および `@@` ショートハンド内。必要なら `{{ }}` も見やすくする |
| 深さ | 初版は **JS 近似**（入れ子言語）で十分。専用 DSL 文法の完全ハイライトは必須ではない |
| 非対象（当面） | VS Code / 他エディタ向けプラグイン化。エコシステム展開は採用時期未定（[wishlist.md](./wishlist.md)） |
| 仕様との整合 | コードフェンス内の hyogen ブロックはハイライト対象外（[dsl.md](./specs/dsl.md) と同様に無視） |

### 出力の空白・改行について

展開後 Markdown の余分な空行（例: `each` 後のリストが項目ごとに開く）は **プレイグラウンド側の CSS / 表示補正では直さない**。ライブラリ本体の出力を、可能な限り手書き Markdown に寄せる（[pipeline.md](./specs/pipeline.md)）。Playground だけ整えて実レンダリングがおかしければ意味がない。
